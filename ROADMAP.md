- [x] Роли участников команды
- [x] Проекты (создание, редактирование, удаление)

- [x] Рил-тайм редактирование документов
- [x] Рил-тайм редактирование таблиц
- [x] Рил-тайм редактирование досок
- [x] Загрузка обычных файлов
- [x] Просмотрщик изображений

## Чаты

- [x] Создание чатов, привязанных к проекту и ко всей команде
- [x] Текстовые сообщения
- [x] Эмодзи-реакции к сообщениям
- [x] Отправка картинок и видео
- [x] Прикрепление остальных файлов
- [x] Отправка голосовых сообщений
- [x] Отображение диаграммы (waveform) у голосовых сообщений
- [ ] Расшифровка голосовых сообщений — *архитектура заложена (`ITranscriptionService`, поле `Transcription`, асинхронный хук, эндпоинт-заглушка), самой расшифровки нет (как и планировалось)*
- [x] Закреплённые сообщения
- [x] Ответы на сообщения
- [ ] Цитирование сообщений — *в сторе есть заготовка (`quoteTarget`), но в UI не подключена: кнопки и вставки цитаты нет*
- [x] Вложенные комментарии (треды, как в Mattermost)
- [x] Созвоны
- [ ] Ссылки на задачи: прикрепление задач к сообщениям через `#` с красивым превью (название + урезанное описание) — *позже; бэкенд заложен (`ITaskResolver`-заглушка, `InlineEntities` в DTO), рендер карточки на UI не сделан*
- [ ] Модель ответов: на первом уровне на сообщения можно отвечать и как в Telegram, и как в Mattermost; на втором уровне — только как в Telegram, а Mattermost-стиль доступен лишь при включённой настройке (рабочее название «режим мазохиста»). — *базовое правило уровней реализовано (`ReplyMode` Thread/Flat), опциональной настройки «режим мазохиста» нет*

## Задачи

- [ ] Название и описание (Markdown с панелью редактора) — *название и описание есть; описание — обычный textarea, Markdown не рендерится и панели редактора нет*
- [x] Два представления: список и Kanban — *есть ещё и таймлайн*
- [x] Назначение исполнителей (в том числе нескольких)
- [ ] Комментарии к задаче с вложением картинок и файлов — *комментарии работают, вложения картинок/файлов к ним не поддерживаются*
- [ ] Метки и статусы задач — *статусы (колонки) есть; метки не доведены: определения меток команды создаются, но привязать метку к задаче нельзя (`LabelIds` игнорируется, ответ всегда пустой)*
- [x] Задачи на уровне проекта и агрегированные — на уровне команды
- [x] Срок (дедлайн) задачи
- [x] Планирование задач на таймлайне

## Календарь

- [ ] События команды (конференции, встречи, сборы) и дедлайны по проектам — *события команды/проекта реализованы полностью; дедлайны задач на календаре не отображаются*
